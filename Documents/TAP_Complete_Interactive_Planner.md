How to read this document
Each card represents one section of the curriculum. The section-level interactive element is designed to
work as the entry point or capstone for that section — giving the user a single experience that connects all
the modules underneath it. It sits above the module-level elements, not instead of them.
SECTION INTENT describes the one thing this element needs to achieve. The element name and
description are the build brief.
Element Types
TOOL
Functional tool — user inputs real data, gets a real output.
SIMULATION
Scenario interaction — user makes choices inside a realistic mock situation.
VISUALISATION
Annotated visual — diagram or map making an attack pattern visible.

SECTION 1
Who You Are & What You Have
What attackers want from you
SIMULATION
4 modules
SECTION INTENT
Before diving into specific threats, the user must first feel the value of what they carry — not as abstract data, but
as a picture of themselves seen through an attacker's eyes.
The Attacker's Shopping List
User is shown a blank profile card with their name at the top. They answer 6 questions about what they own,
use, and share online — accounts, devices, payment methods, location habits, identity documents, social
connections. As each answer is entered, the card fills in from the attacker's perspective: not 'email address'
but 'account reset vector', not 'home address' but 'physical access point', not 'phone number' but 'SIM swap
target'. By the end the user sees a complete picture of themselves as a target — built entirely from ordinary
information they share without thinking.
SECTION 2
Social Engineering & Manipulation Tactics
Exploiting human psychology to steal from you
SIMULATION
10 modules
SECTION INTENT
Ten modules of psychological tactics are only useful if the user can recognise them arriving simultaneously in a
single real attack. The section-level element collapses the full toolkit into one scenario.
The Perfect Attack
User receives a single realistic social engineering scenario — a phone call transcript from a 'bank fraud
team' — and must identify every psychological tactic embedded in it. The script deliberately layers multiple
triggers: authority (bank officer title), urgency (transaction happening now), fear (account will be frozen),
reciprocity (we are trying to help you), and commitment (you already gave us your name, just confirm the
OTP). Each tactic is hidden in plain sight. After the user marks what they spotted, the full annotated
breakdown reveals every lever — including the ones they missed. The goal is not to pass a test but to see
how many tactics can co-exist in a two-minute conversation.

SIMULATION

SECTION INTENT
Fourteen scam types share a common operating logic underneath their different disguises. The section element
reveals that structure so users recognise new scams they have never seen before.
The Scam Pattern Matcher
User is presented with 5 unfamiliar scam scenarios — formats they are unlikely to have seen before, drawn

SECTION 4
Deceptive Communication
How messages and content lie to you
TOOL
6 modules
SECTION INTENT
Six modules cover the full spectrum of deceptive communication — email, SMS, voice, websites, QR codes,
deepfakes, and fake credentials. The section tool gives the user a single moment of applied verification across all
formats.
The 60-Second Verification Kit
User is given a deceptive communication scenario — a message, a link, a call, or a piece of content — and
the tool walks them through a 4-step rapid verification process applicable to any format: check the source
(sender, domain, caller ID), check the request (what is being asked and why now), check the channel (is this
the expected way this organisation communicates), check independently (find the official contact yourself,
do not use the one provided). The tool generates a structured verification checklist tailored to the specific
communication type selected. Usable as a real-time reference, not just a learning exercise.
SECTION 5
AI Bias & Algorithmic Influence
How AI systems manipulate and mislead you
VISUALISATION
5 modules
SECTION INTENT
Five modules cover distinct AI failure modes. The section visual connects them into a single coherent picture of
what it means to live inside algorithmically mediated reality.
The Invisible Hand Map
An illustrated diagram showing a single person moving through one ordinary day — morning news feed, job
application, insurance renewal, social media scroll, evening recommendation algorithm. At each touchpoint,
an AI system is making a decision about them or showing them something. Each decision point is annotated

confidence in the insurance pricing model. No interaction needed — the map makes visible what is normally
invisible, and asks one question at the end: how many of these decisions did you know were being made?

SECTION 6
Dark Patterns & Manipulative Design
How apps and websites trick you into giving away data and money
SIMULATION
11 modules
SECTION INTENT
Eleven dark patterns are easier to absorb individually than collectively. The section element forces the user to
encounter all of them in a single product experience — the way they actually appear in the real world.
The Dark Pattern Gauntlet
User signs up for a fictional free trial service and must complete the full onboarding and then cancel. The
entire flow is engineered with one dark pattern per screen: pre-ticked consent boxes, confirm-shaming on
the reject button, hidden pricing revealed only at checkout, artificial urgency countdown, misleading free trial
terms, roach motel cancellation path, fake social proof, manufactured scarcity, nudged defaults, emotional
manipulation in the retention popup, and a final obfuscated confirmation screen. Each pattern is named in
real time as the user encounters it. By the end they have experienced the full section in under 3 minutes —
and will recognise every pattern the next time they see it in the wild.

How commercial systems deceive, profile, and exploit you for profit
VISUALISATION
7 modules
SECTION INTENT
Seven modules expose different commercial manipulation mechanisms. The section visual shows how they
function as a single coordinated system rather than isolated tactics.
The Manipulation Stack
A layered diagram — like a technology stack — showing the full commercial manipulation architecture sitting

layer: targeting engine (lookalike audiences, vulnerability scoring, emotional state detection). Next layer:

appearing as a simple, helpful product recommendation. Each layer is annotated with the module from this
section that covers it in depth. The ad at the top is revealed to be the output of a 5-layer manipulation
pipeline the user never sees.

SECTION 9
Financial & Economic Exploitation
How money is taken or how you are exploited for profit
SIMULATION
3 modules
SECTION INTENT
Three modules cover payment fraud, account takeover, and attention monetisation. The section simulation
connects financial exploitation to the moment it actually happens — under pressure, in real time.
The 10-Minute Window
User is placed in a high-pressure scenario: they have just noticed an unauthorised transaction on their
account while simultaneously receiving a phone call from someone claiming to be their bank's fraud team.
They must manage both simultaneously — responding to the caller while deciding what actions to take on
their device. The caller uses authentic-sounding language and accurate account details (obtained via prior

their choices — and the key insight is revealed: the call and the transaction are both part of the same attack,
and the caller is the attacker.
SECTION 10
Harassment, Abuse & Harm
Attacks targeting your safety, reputation, and wellbeing
TOOL
5 modules
SECTION INTENT
Five modules cover different forms of digital harm. The section tool recognises that someone encountering this
section may be experiencing one of these situations right now — and needs a way in, not a lecture.
The Right Help Finder
User selects the situation closest to theirs from a plain-language list: someone is sending me threatening
messages, intimate images of me have been shared without consent, I am being tracked or monitored by
someone I know, I am being harassed by a group online, or someone is trying to damage my reputation.
Based on their selection, the tool immediately surfaces the 3 most relevant actions: the right platform
reporting path, the right law enforcement contact for their country, and the right support organisation. No
preamble, no module list — just the next step. Designed to be the first thing someone finds when they need
help, not the last.

SECTION 11
Psychological & Behavioural Manipulation
How systems exploit how humans think and behave
SIMULATION
6 modules
SECTION INTENT
Six modules cover the cognitive and behavioural vulnerabilities that make manipulation possible. The section
simulation shows the user their own vulnerability profile — not as a flaw but as a map.
Your Vulnerability Window
User completes a short 12-question behavioural assessment: not a personality test, but a situational one —
when do you check your phone first, how do you respond under time pressure, what triggers you to share
content, when do you trust authority figures, how do you behave when you feel you have already invested in
something. The output is not a score but a personalised vulnerability window: the 2 or 3 specific conditions
under which this user is most susceptible to manipulation, with the specific attack types that target each
condition. Framed as self-knowledge rather than weakness — understanding your window is the first step to
closing it.
SECTION 12
Specialised Threats & Vulnerable Groups
High-risk situations and specific vulnerabilities
SIMULATION
9 modules
SECTION INTENT
Nine modules cover groups and contexts with elevated risk. The section simulation asks the user to step outside
their own experience and see risk through another person's eyes.
The Other Person's Screen
User selects a profile different from their own from a set of six: a 68-year-old managing a pension, a
14-year-old on gaming platforms, a recent migrant worker, a person in financial distress, a domestic abuse
survivor, or a journalist in a high-risk country. They are then shown a day in that person's digital life — the
messages they receive, the platforms they use, the decisions they face. At 3 points in the day a threat
arrives. User must identify it and choose the response available to that person given their specific
constraints. Builds protective awareness for people around the user — not just for themselves.
SECTION 13
When It Happens to You
Recovery and harm mitigation
TOOL
9 modules

SECTION INTENT
Nine modules cover recovery across every type of digital harm. The section tool is the one the user needs before
anything has happened — because building a response plan in advance is the difference between acting and
freezing.
Your Personal Response Plan
User spends 5 minutes building a personalised incident response card before anything goes wrong. They
record: their bank's 24-hour fraud line, their most important account recovery emails and phone numbers,
their country's cybercrime reporting portal, one trusted person to call in a crisis, and the 3 accounts that
would cause the most damage if compromised. The tool produces a one-page formatted card — printable or
saveable — with their specific contacts and the correct sequence of actions for the 3 most likely incident
types. The card exists so they do not have to think clearly in the moment when thinking clearly is hardest.

Country-Specific Threats, Scams & Resources
Localised risks and help based on where you live
TOOL
5 modules
SECTION INTENT
Five country modules cover localised risks and reporting paths. The section tool gives every user a single
personalised reference point regardless of which country module is most relevant to them.
Your Country Kit
User selects their country or the country they are currently in. The tool assembles a single-page
personalised kit: the top 3 scam types currently active in that country (based on documented regional fraud

SECTION 15
Your Digital Protection Habits
Simple practices that make you significantly harder to target
TOOL
6 modules
SECTION INTENT
Six modules cover practical protection across devices, accounts, apps, networks, data, and behaviour. The
section tool synthesises everything into a single, honest picture of where the user actually stands.
Your Protection Scorecard
User completes a 15-point audit across the six habit domains from this section: 3 questions each on
devices, accounts, apps, network, data settings, and daily behaviour. Each question is binary — done or not
done. No partial credit. The output is a scorecard showing current protection level across all six domains
with a single overall readiness rating. Critically: the tool does not generate an overwhelming fix list. It
identifies the one change in each domain that would have the highest protective impact for the lowest effort
— giving the user 6 specific next actions, not 15. Designed to close the gap between knowing and doing.

MODULE 1
Your Identity & Personal Information
TOOL
Identity Value Calculator — The user inputs a few everyday data points (name, city, phone number, email,
national ID type). As each field is filled, a running 'dark web value' counter updates in real time, showing the
estimated monetary worth of that data on breach markets. Final screen shows the combined profile value and
a breakdown of which data types are most exploitable. Goal: make the abstract feel viscerally real.
MODULE 2
Your Passwords & Account Access
TOOL
Password Crack-Time Estimator — User types any password into a local (no-network) input field. The tool
instantly shows estimated crack time using brute force, dictionary attack, and credential stuffing approaches.
A live strength meter shifts from red to green. A second panel shows what happens to every linked account if
just one password is reused. Entirely client-side — no data leaves the browser.
MODULE 3
Your Money & Financial Assets
SIMULATION
Spot the Fake Message — Two side-by-side messages appear on screen: one is a real bank alert, one is a
fraudulent WhatsApp message mimicking it. The user must tap the elements that give away the fake (sender
ID, grammar, link domain, urgency language). Each correct tap reveals an explanation. Replayable with new
message pairs drawn from a set of 6 real-world scam templates.
MODULE 4
Your Location & Physical Safety
TOOL
EXIF Metadata Reader — User uploads any photo from their device. The tool reads and displays all
embedded EXIF data: GPS coordinates (plotted on a map), device model, timestamp, camera settings. If
GPS data is present, a warning banner appears explaining exactly what a stranger could determine from that
single image. A 'Strip Metadata' button produces a cleaned copy with all EXIF removed for download. Entirely
client-side.
